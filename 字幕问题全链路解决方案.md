# 字幕问题完整修复总结 - 字幕问题全链路解决方案

## 问题根因（已100%确认）

### 根本原因：后端翻译服务的 bug

**后端在发送 caption_update 事件时，从 'interim' 状态切换到 'stable' 状态时会发送截断的文本。**

### 实际案例

#### 案例 1：seg_6b1e5e3213e2

**完整事件序列**：
```
transcript.partial:  114 字符（完整）
caption_update interim: 114 字符（完整）
caption_update stable:   78 字符 ← 截断了 36 字符！
```

**文本对比**：
- **完整**（149字符）：`an extremely resource-intensive research topic, can we also scale the science to allow the community to drive the collective progress of this topic?`
- **interim**（114字符）：`an extremely resource-intensive research topic, can we also scale the science to allow the community to drive the`
- **stable 截断**（78字符）：`an extremely resource-intensive research topic, can we also scale the science` ← **丢失 71 字符！**

#### 案例 2：seg_5545acea3218

```
transcript.partial: 38 字符 "Some of you might have already seen it"
caption_update:      6 字符（多次更新，越来越短）
```

### 影响范围

通过日志分析，发现 **20 个 segment** 受影响，累计丢失文本超过数百字符。

### 修复 4：caption_update 文本保护（核心修复）✅
**文件**：`apps/desktop/src/shared/caption-store.ts:selectCaptionUpdateSourceText`

```typescript
function selectCaptionUpdateSourceText(previousLine, event) {
  const newText = event.source.full_text;
  
  if (!previousLine) {
    return newText;
  }

  // 关键修复：无论 state 是什么，都保护更长的文本
  if (previousLine.sourceText.length > newText.length + 5) {
    console.warn("[caption_update] 检测到文本异常缩短", {
      segment_id: event.segment_id,
      state: event.state,
      prev_len: previousLine.sourceText.length,
      new_len: newText.length,
      shrink: previousLine.sourceText.length - newText.length,
      action: "保留旧文本"
    });
    return previousLine.sourceText;
  }

  return newText;
}
```

**修复前的问题代码**：
```typescript
if (event.state === "final" || !previousLine) {
  return event.source.full_text; // ❌ 直接覆盖，不保护
}
```

**效果**：
- ✅ 防止 caption_update 的截断文本覆盖完整文本
- ✅ 无论 state 是 interim/stable/final 都生效
- ✅ 添加告警日志，便于追踪后端问题
