# 3 天极速开发：开源 AI 同传工作站技术选型与实现报告（国内友好版）

## 当前状态说明

本文是早期 **LiveKit 优先** 的 3 天快速方案，用于保留选型背景和后续远程房间方向。当前仓库已经先接通 **Electron/Web Audio PCM -> FastAPI WebSocket -> Python Agent -> Electron 悬浮字幕窗** 的本地桌面链路。

因此：

```text
当前主链路：Desktop 8766 WebSocket 本地同传
后续候选：LiveKit / WebRTC 远程音频传输
```

不要把本文中的 LiveKit 排期理解为当前已接通状态。`livekit_agent.py` 仍是传输适配占位，当前真实桌面链路使用 `/v1/realtime/sessions/{session_id}` 和 `/v1/caption/events`。

## 背景约束

在 3 天开发周期，以及“个人开发者、开源免费、国内网络友好”的约束下，系统架构应采用：

- 成熟的 WebRTC 编排框架；
- 国内友好的单点模型服务；
- 可快速替换的 Python Agent 编排层；
- 字幕优先、TTS 可选的最小闭环。

推荐基调：

```text
Next.js 前端
  -> LiveKit 实时音视频网关
  -> Python Agent 后端推理编排
  -> FunASR / DeepSeek / edge-tts
```

## 核心技术栈

### 实时音视频流转：LiveKit

LiveKit 是开源 WebRTC 实时流媒体服务器，提供前后端 SDK 和 React 组件。

选型理由：

- 3 天内手写 WebSocket、音频重采样、丢包重传和静音检测成本过高。
- LiveKit 自带房间、音轨、数据通道和前端组件。
- `livekit-agents` 可以作为 Python Agent 的基础框架。
- 可先利用 LiveKit Cloud 免费额度快速起步。

### ASR：FunASR

FunASR 是通义实验室开源语音识别模型，支持多语种、流式识别、标点恢复、时间戳和热词能力。

选型理由：

- 国内网络友好。
- 中文和中英混合识别能力强。
- 适合技术分享中的热词和专有名词。
- 可通过 WebSocket 流式服务接入 Python Agent。

### 翻译模型：DeepSeek

DeepSeek-V3 具备高性价比和较强推理能力，接口兼容 OpenAI API。

选型理由：

- 国内可用性较好。
- 接入成本低。
- 可以复用 OpenAI-compatible SDK。
- 适合在提示词中注入术语表和上下文。

### TTS：edge-tts

edge-tts 是 Python 开源模块，可调用微软 Edge 在线语音能力。

选型理由：

- 免费、零配置、无需 API Key。
- 中文语音自然度足够支撑 MVP。
- 作为可选输出，不放在首版关键路径。

## 架构与数据流

本方案采用 Agent 架构，前后端职责解耦。

```text
客户端（Next.js）
  -> 使用 @livekit/components-react 接入麦克风或标签页音频
  -> 推送音频到 LiveKit Room

LiveKit
  -> 转发 WebRTC 音频流
  -> 提供数据通道发送字幕事件

Python Agent
  -> 接收音频帧
  -> 调用 FunASR 进行实时转写
  -> 调用 DeepSeek 翻译为中文
  -> 通过修正窗口生成字幕补丁
  -> 可选调用 edge-tts 生成译文语音
```

## 自定义适配器方案

FunASR 和 edge-tts 不是 LiveKit 官方商业插件，因此需要写自定义适配器。

### 自定义 ASR

建议继承或适配 `livekit.agents.stt.STT` 语义：

- 接收 LiveKit 音频帧。
- 按 FunASR 流式服务需要的切片格式发送，例如 `chunk_size="5,10,5"`。
- 将 FunASR 返回结果转换为内部 `TranscriptSegment` 或 LiveKit `SpeechEvent`。

### 自定义 TTS

建议继承或适配 `livekit.agents.tts.TTS` 语义：

- 在 `synthesize` 中调用 `edge_tts.Communicate`。
- 逐块产出音频字节流。
- 交给 LiveKit 或前端播放器消费。

## 72 小时开发排期

### 第 1 天：基础设施和 LiveKit 通道

上午：

- 注册 LiveKit Cloud。
- 获取 `LIVEKIT_URL`、`LIVEKIT_API_KEY`、`LIVEKIT_API_SECRET`。
- 初始化 Next.js 前端。
- 安装 `@livekit/components-react`。

下午：

- 实现前端 LiveKit 房间连接。
- 渲染麦克风采集按钮。
- 初始化 Python 环境。
- 安装 `livekit-agents[silero,turn-detector]`、`python-dotenv`、`websocket-client`。

晚上：

- 运行基础 Python Agent。
- 确认前端说话时，后端能收到音频帧事件。

### 第 2 天：ASR、翻译和 TTS

上午：

- 启动 FunASR 实时服务，例如 `funasr-server --device cuda --port 10095`。
- 在 Python Agent 中实现 FunASR 适配器。
- 将音频帧发送到 FunASR，获取流式文本。

下午：

- 接入 DeepSeek API。
- 设置系统提示词，使模型只做同声传译，不输出额外解释。
- 将转写文本翻译为中文。

晚上：

- 实现 edge-tts 适配器。
- 把译文转成音频字节流。
- 作为可选语音输出推回前端。

### 第 3 天：字幕状态机和体验优化

上午：

- 通过 LiveKit 数据通道或 RPC 推送字幕事件。
- 前端接收源语言字幕、中文翻译和提交事件。

下午：

- 将字幕分为“已确认”和“识别中/翻译中”两类。
- 已确认字幕高对比度展示。
- 未稳定字幕使用浅色或半透明展示。
- 根据 FunASR 的中间/最终结果驱动状态。

晚上：

- 做断线重连和异常捕获。
- 处理 FunASR WebSocket 断开、DeepSeek 请求失败、edge-tts 失败等情况。
- 前端可部署到 Vercel 或 Netlify。
- Python Agent 可部署到腾讯云或阿里云轻量服务器。

## 成本和性能优化

### VAD 前置

尽量在 LiveKit 或 Agent 层启用 Silero VAD。只有检测到说话时才把音频送入 FunASR 和 DeepSeek。

收益：

- 降低服务器带宽。
- 减少 ASR 推理。
- 减少 LLM token 消耗。

### TTS 缓存

对于频繁出现的短语，例如“你好”“谢谢”，可以在 Python 内存中维护 LRU 缓存。

命中缓存时直接返回音频，避免重复调用 edge-tts。

### 成本优势

该架构中：

- 前端托管可以免费起步。
- FunASR 开源免费。
- edge-tts 免费。
- 主要变量成本来自 DeepSeek token。

因此它适合个人开发者长期维护、试错和快速迭代。
