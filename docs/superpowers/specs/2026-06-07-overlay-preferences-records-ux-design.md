# EchoSync 字幕面板、偏好设置与记录查看 UX 设计

## 背景

本设计来自 2026-06-07 对 EchoSync 当前前端的集中审查。用户反馈的核心不是单个按钮样式，而是几个界面层级混在一起：

- 悬浮字幕窗的默认态、hover 态、设置态、锁定态切换不流畅。
- 上下工具栏职责混乱，有些按钮无效或事件不清。
- 分区对照和逐句对照没有形成稳定的阅读模型。
- 首页堆叠了预览、免费时长、无效入口和不可修改语言。
- 偏好设置缺少模型、术语、字幕窗口、记录隐私的清晰分组。
- 会议记录查看面板信息密度过高，长文本溢出，左右栏自适应不足。

目标是建立一个可持续扩展的信息架构，而不是继续给现有界面补零散样式。

## 产品原则

1. 首页只做启动前的高频选择：输入源、语言、质量/方案、开始同传。
2. 悬浮字幕窗只服务实时观看：字幕内容优先，鼠标进入才显示控制。
3. 偏好设置做长期管理：模型、术语库、默认语言、字幕窗口、记录隐私。
4. 同传中只保留现场必需的轻量控制：开始/结束、输入切换、显示模式、语言、方案。
5. 会议记录页只做复盘：播放、片段浏览、摘要、导出、编辑。
6. 所有控件必须有真实事件或明确禁用态，不能保留看似可用但无效的图标。

## 悬浮字幕窗

### 状态机

悬浮窗使用显式状态，而不是只靠 CSS `:hover`：

- `default`：低干扰字幕态。字幕可读，背景透明，工具栏隐藏，文本不可选。
- `controls`：鼠标进入或快捷键唤醒后的操作态。顶部工具栏和底部会话栏淡入。
- `settings`：菜单或设置浮层打开。即使鼠标短暂离开，也保持操作态。
- `pinned`：锁定/驻留态。窗口可交互，保留解锁入口，避免误拖动和误关闭。
- `dragging`：拖拽或 resize 期间，禁止 collapse。

交互时序：

- 鼠标进入后约 160-220ms 进入 `controls`，避免扫过窗口误触发。
- 鼠标离开后约 650-900ms 才回到 `default`，期间鼠标回来则取消 collapse。
- 菜单打开时不 collapse。
- 锁定后显示明确的“解锁字幕”入口，不能只靠快捷键。

字幕文本默认 `user-select: none`，防止设置菜单、下拉框和字幕文本选择互相干扰。

### 顶部工具栏

顶部工具栏只放窗口与字幕展示控制：

- 分区 / 逐句：打开显示模式菜单。
- 锁定字幕：进入锁定/驻留态。
- 置于顶层：切换 always-on-top 或 pinned 行为。
- 设置：打开字幕/窗口设置。
- 最小化：隐藏悬浮窗。
- 关闭：关闭字幕窗。

删除无意义图标。图标尺寸、描边、圆形背景、hover/selected 状态必须统一。

### 底部会话栏

底部会话栏只放会话控制：

- 输入切换：麦克风 / Windows 系统声音。
- 开始 / 结束。
- 计时。
- 模型或方案：通用、实时优先、准确复盘等。
- 语言设置。
- 内容显示：原文 / 译文 / 双语。

这些控件在第一阶段可以先接现有事件或禁用态；后续接真实模型/语言配置。

### 分区对照

分区对照是上下双轨：

- 上区显示原文，下区显示译文。
- 两区各自靠左、靠上排版，不居中。
- 两区互不干扰，各自像独立窗口一样向上滚动。
- 内容占满后向上滚出。
- 新内容淡入、旧内容上移，动画使用 transform/opacity，不改变窗口尺寸。

### 逐句对照

逐句对照是默认模式：

- 原文在上，译文在下。
- 实时内容先连续增长，不立即强行断句。
- 达到约三行或稳定阈值后，沉淀为一组对照块。
- 已沉淀块上滚时保持源译对应关系。
- 译文迟到或修订时，只影响尾部或当前块，不整窗跳动。

## 首页

首页应移除以下内容：

- `免费 1 小时` 顶栏提示。
- 当前显眼的 `实时字幕` 顶栏入口。
- 首页大块字幕预览。
- `打开字幕窗预览` 主操作。
- 右上角厚重圆形窗口按钮。

首页保留：

- 音频源：麦克风 / Windows 系统声音。
- 目标语言：必须可修改。
- 质量或方案：实时优先 / 均衡 / 准确复盘。
- 开始同传。
- 状态提示。
- 会议记录、偏好设置作为低调入口。

## 偏好设置

偏好面板拆成清晰分组。

### 常用

- 默认音频源。
- 默认源语言 / 目标语言。
- 默认质量方案。
- 默认字幕显示方式。

### 模型

支持多个模型配置，而不是一个静态供应商开关：

- ASR 模型配置。
- 翻译模型配置，可配置多个，例如 GPT-4o、GPT-4o mini、DeepSeek、服务端默认。
- TTS 模型配置。
- 每个配置包含：名称、供应商、模型 ID、凭证来源、启用状态、连接测试。
- 首页只显示当前方案，不做复杂编辑。

### 术语

完整术语库管理放在偏好面板：

- 导入 CSV/TXT/JSON 或粘贴导入。
- 术语库列表：名称、语言方向、条目数、启用状态、更新时间。
- 支持启用/停用。
- 同传中只保留“临时术语快速添加”。
- 复盘页可展示命中的术语，但不负责长期管理。

### 字幕与窗口

- 字号、颜色、描边、窗口阴影、透明度。
- 默认显示模式：逐句对照 / 分区对照。
- hover 延迟、collapse 延迟。
- 锁定、置顶、最小化行为。

### 记录与隐私

- 是否保存原始音频。
- 是否保存双语记录。
- 自动清理周期。
- 诊断日志导出。

## 会议记录查看

### 信息架构

顶部区域：

- 记录标题。
- 返回 / 关闭。
- 导出 TXT / SRT / Markdown。
- 摘要生成状态。

正文区域：

- 左侧：音频播放器 + 片段列表。
- 右侧：摘要、关键词、主题、术语建议。

窄窗口：

- 自动变为上下布局。
- 左侧片段和右侧摘要各自滚动，不互相撑高。

### 文本排版

- 原文、译文必须 `overflow-wrap: anywhere`，长词和英文句子不能溢出卡片。
- 片段卡片自适应高度，但设置合理最大高度。
- 当前播放片段高亮。
- 摘要侧栏按块分组，不把所有标签、主题、建议混成一团。

## 实施阶段

### 第一阶段：悬浮字幕窗基础体验

- 调整 hover/collapse 时序。
- 文本不可选。
- 顶部工具栏重组为：显示模式、锁定、置顶、设置、最小化、关闭。
- 底部会话栏重组为：输入源、开始/结束、时间、方案、语言、内容显示。
- 增加锁定后的“解锁字幕”入口。
- 调整 `sentencePair` / `zonedPair` 的菜单与文案。
- 验证 default、controls、menuOpen、pinned 四个状态。

### 第二阶段：首页和偏好设置

- 删除免费时长、字幕预览、预览入口。
- 目标语言改为可选控件。
- 偏好设置改为分组结构。
- 新增模型配置和术语库管理的 UI 骨架。

### 第三阶段：会议记录

- 重排记录详情页。
- 修复片段长文本溢出。
- 摘要侧栏分块和独立滚动。
- 窄窗口自适应。

### 第四阶段：真实后端闭环

- 多模型配置持久化。
- 术语库导入与 Agent glossary 同步。
- 语言设置接入真实会话参数。
- 记录隐私设置持久化。

## 验收

- 鼠标进入字幕窗时工具栏稳定出现，不需要只靠快捷键。
- 鼠标离开后延迟淡出，菜单打开时不消失。
- 锁定态有明确解锁入口。
- 字幕文本不能被鼠标选中。
- 顶部和底部工具栏职责清晰，无无效图标。
- 分区对照上下双区靠左靠上，内容满后自然上滚。
- 逐句对照约三行后沉淀为对照块。
- 首页无免费时长，无大块预览，目标语言可改。
- 偏好面板包含模型和术语管理入口。
- 会议记录长文本不溢出，摘要和片段各自滚动。
