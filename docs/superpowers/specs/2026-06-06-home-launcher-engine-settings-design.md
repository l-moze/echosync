# EchoSync 首页同传启动器与引擎设置设计规范

## 目标

把 EchoSync Desktop 首页从“模型控制台”收敛为“同传启动器”。首页只服务用户开始同传、确认音频输入、确认字幕窗效果和选择用户可理解的质量模式；模型、供应商、Mock、WebSocket、日志等工程配置统一进入设置页的“引擎与性能”或“开发者”区域。

## 调研结论

Teams、Google Meet、Zoom 的实时字幕体验都围绕“字幕开关、字幕语言、目标语言、字幕样式、字幕位置”组织，不让普通用户选择 ASR 或翻译模型。Material Design 的设置模式也强调优先展示重要设置，次要或大量设置放入单独页面。EchoSync 应沿用同一产品心智：用户来这里是为了给当前音频挂字幕，不是为了调模型。

## 核心产品原则

1. **首页做同传启动器**

   首页只回答四件事：听哪路声音、翻译成什么语言、字幕长什么样、现在能不能开始。

2. **设置页做引擎配置**

   ASR、翻译、TTS、端到端模型、fallback、Mock、日志都不是首页主任务。它们进入设置页，一级菜单命名为“引擎与性能”，不使用“模型路由”这类工程语言。

3. **字幕体验是视觉中心**

   首页最大的视觉资产应是双语字幕预览，而不是 provider 卡片。字幕预览用于展示双语层级、半透明背景、可读字号和覆盖效果。

4. **正常状态轻量，异常状态展开**

   首页默认只显示一行状态摘要。只有音频无输入、字幕窗不可用、同传服务未连接、引擎不可用时，才展开诊断详情。

5. **开发能力不污染普通体验**

   Mock、WebSocket 地址、事件调试、延迟日志进入“开发者”。普通首页不出现 `Mock`、`ASR`、`TTS`、`Agent`、`模型路由`、`provider` 等词。

### 首页保留的信息

| 信息 | 首页表达 | 说明 |
|---|---|---|
| 产品状态 | `已就绪` / `需要检查` / `同传中` | 放在顶栏右侧，避免大块状态卡。 |
| 音频源 | `Windows 系统声音`、`麦克风` | 使用下拉或两段式控件，不用大卡片。 |
| 语言方向 | `English → 中文` | 后续支持更多语言时改为下拉。 |
| 质量模式 | `实时优先`、`均衡`、`准确复盘` | 用户语言，不显示具体模型。 |
| 主操作 | `开始同传` | 页面唯一高权重按钮。 |
| 次操作 | `打开字幕窗预览` | 文字按钮或浅色按钮。 |
| 字幕预览 | 双语浮层预览 | 首页视觉中心。 |
| 状态摘要 | `已就绪 · 系统声音有输入 · 字幕窗可用 · 引擎自动` | 正常态一行展示。 |

### 首页禁止出现的信息

| 禁止项 | 原因 | 应放位置 |
|---|---|---|
| `Mock` | 开发者语义，会降低产品完成度 | 设置 / 开发者 |
| `ASR` | 模型层术语，普通用户不需要 | 设置 / 引擎与性能 |
| `TTS` | 缩写不够产品化 | 设置 / 引擎与性能，用“语音播报”表达 |
| `模型路由` | 实现语言 | 设置 / 引擎与性能 |
| `后端默认` | 工程配置语言 | 首页显示“引擎自动” |
| `Voxtral + DeepSeek` | 可作为高级摘要，不应常驻首页 | 设置详情 |
| WebSocket / 事件调试 | 开发调试 | 设置 / 开发者 |

| 工程文案 | 首页替代表达 |

| Agent 已读取能力 | 同传服务已连接 |

首页最多保留一行轻技术摘要：

```text
引擎：自动 · 当前为低延迟方案
```

```text
语音识别：Voxtral Realtime
翻译：DeepSeek
备用：FunASR
```

## 引擎与性能页设计

引擎设置不使用卡片网格。推荐使用分组表单或列表行，便于未来扩展很多模型。

```text
引擎与性能

自动模式
  实时优先    均衡    准确复盘

当前方案
  语音识别    自动（Voxtral Realtime）     可用
  翻译        自动（DeepSeek）             可用
  语音播报    关闭                         未启用
  端到端      关闭                         未启用

失败切换
  语音识别失败时切换到 FunASR
  翻译失败时保留原文并提示
```

如果后续接入大量模型，采用搜索式选择器：

```text
选择语音识别引擎
搜索模型或供应商...

自动推荐
Voxtral Realtime      云端 · 低延迟 · 英语强
FunASR                本地 · 隐私 · 中文强
Azure Speech          云端 · 企业 · 多语言
```

## 视觉方向

### 首页

- 背景：简洁纯白或极浅灰为主；如果采用深色半透明面板，只用于字幕预览和悬浮字幕，不让背景抢内容。
- 主面板：一个，不要多个大卡片。
- 主按钮：唯一高权重，建议蓝色或青蓝色。
- 字幕预览：尺寸要大，双语层级清楚；英文较小，中文较大。
- 状态摘要：细字、低对比、单行。
- 模型与状态诊断：默认收起，异常才展开。

### 悬浮字幕预览

- 半透明深色背景，轻微毛玻璃。
- 8-12px 圆角，弱边框。
- 中文译文为主字号，原文为次字号。
- 鼠标聚焦时出现轻工具栏；默认不显示复杂控制。

### 控件风格

- 音频源和目标语言用下拉行，不用模型卡片。
- 质量模式用三段式 segmented control。
- 开始同传用单一主按钮。
- 设置入口用轻量图标或文字按钮。

## 状态展示策略

### 正常态

```text
已就绪 · 系统声音有输入 · 字幕窗可用 · 引擎自动
```

### 异常态

异常时显示一条明确提示和修复入口：

```text
同传服务未连接
请先启动本地同传服务，或检查 8766 端口。
```

```text
系统声音暂无输入
请播放视频，或切换到麦克风。
```

```text
当前引擎不可用
请在“引擎与性能”中检查密钥或切换自动模式。
```

## Active 与 Finished 状态

本规范主要约束 Idle 首页，但 Active 和 Finished 也要延续同样语言：

- Active 不叫“会话驾驶舱”，可以叫“正在同传”。
- Active 的右栏不常驻展示模型名，只显示“质量模式：均衡”“延迟：xxx ms”“字幕窗：已打开”。
- Finished 保留复盘能力，重点是原始音频回放、双语记录、导出，不展示模型配置。

## 非目标

- 本轮不重做悬浮字幕窗口交互细节。
- 本轮不新增真实模型供应商适配。
- 本轮不改变 Agent capabilities 协议。
- 本轮不移除开发者能力，只是从普通首页迁移到开发者设置。

## 验收标准

1. 首页不出现 `Mock`、`ASR`、`TTS`、`模型路由`、`provider`、`后端默认` 等开发术语。
2. 首页不出现模型选择卡片网格。
3. 首页只有一个视觉主按钮：`开始同传`。
4. 首页保留音频源、目标语言、质量模式、字幕窗预览、状态摘要。
5. 字幕预览是首页最大的信息视觉区域之一。
6. 模型选择统一进入“设置 / 引擎与性能”。
7. Mock、WebSocket、延迟日志、事件调试统一进入“设置 / 开发者”。
8. 正常状态只显示一行摘要；异常状态才展开诊断。
9. 视觉上避免后台管理系统风格：减少卡片数量、弱边框、充足留白、克制强调色。

## 实施建议

第一步只改 Idle 首页，把模型卡片迁走，保留现有能力预检逻辑但改成产品化摘要。

第二步新增设置抽屉或设置页中的“引擎与性能”，把现有 `ASR_PROVIDER_OPTIONS`、`TRANSLATION_PROVIDER_OPTIONS` 和延迟模式搬进去。

第三步把 Mock 入口隐藏到开发者设置。开发者模式可以先通过布尔状态或后续配置开关控制，不影响普通用户默认体验。

第四步补充 focused tests：检查首页文案不含开发术语，检查引擎设置页仍能选择 provider，检查启动 payload 不受 UI 重构影响。
