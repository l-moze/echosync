# EchoSync 首页同传启动器、偏好设置与会议记录设计规范

## 目标

把 EchoSync Desktop 从“模型控制台”收敛为三个清晰产品面：主控制面板负责开始同传，字幕窗口负责覆盖式实时字幕，会议记录负责回看、校对、播放和导出。偏好设置只保留少量默认偏好和质量模式；模型、fallback、Mock、WebSocket、原始日志等工程能力只进入高级或开发者入口，不能污染首页、字幕窗口和会议记录。

## 调研结论

Teams、Google Meet、Zoom 的实时字幕体验都围绕“字幕开关、字幕语言、目标语言、字幕样式、字幕位置”组织，不让普通用户选择 ASR 或翻译模型。Material Design 的设置模式也强调优先展示重要设置，次要或大量设置放入单独页面。EchoSync 应沿用同一产品心智：用户来这里是为了给当前音频挂字幕，不是为了调模型。

- Google Meet translated captions：https://support.google.com/meet/answer/10964115

## 核心产品原则

1. **首页做同传启动器**

   首页只回答四件事：听哪路声音、翻译成什么语言、字幕长什么样、现在能不能开始。

2. **偏好设置保持简单**

   设置不是越全越好。普通偏好设置只暴露默认音频源、默认语言、字幕外观、同传质量、记录与隐私。引擎、fallback、Mock、WebSocket、原始日志不出现在普通设置第一屏。

3. **字幕体验是视觉中心**

   首页最大的视觉资产应是双语字幕预览，而不是 provider 卡片。字幕预览用于展示双语层级、半透明背景、可读字号和覆盖效果。

4. **正常状态轻量，异常状态展开**

   首页默认只显示一行状态摘要。只有音频无输入、字幕窗不可用、同传服务未连接、引擎不可用时，才展开诊断详情。

5. **开发能力不污染普通体验**

   Mock、WebSocket 地址、事件调试、延迟日志进入“开发者”。普通首页不出现 `Mock`、`ASR`、`TTS`、`Agent`、`模型路由`、`provider` 等词。

6. **会议记录是独立产品面**

   会议记录不属于偏好设置，也不属于字幕窗口设置。它是“同传之后”的复盘入口，重点是记录检索、原始音频回放、原文/译文对齐、字体调整和导出。

## 首页信息架构

### 页面结构

```text
┌────────────────────────────────────────────┐
│ EchoSync                         已就绪  设置 │
├────────────────────────────────────────────┤
│                                            │
│              实时双语字幕                  │
│      为当前音频生成低延迟字幕覆盖层          │
│                                            │
│  音频源        Windows 系统声音        ▾     │
│  目标语言      English → 中文          ▾     │
│  质量模式      实时优先｜均衡｜准确复盘       │
│                                            │
│              [ 开始同传 ]                   │
│           打开字幕窗预览                    │
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │  The speaker is explaining...         │  │
│  │  演讲者正在解释实时字幕的工作方式。       │  │
│  └──────────────────────────────────────┘  │
│                                            │
│  已就绪 · 系统声音有输入 · 字幕窗可用 │
└────────────────────────────────────────────┘
```

### 首页保留的信息

| 信息 | 首页表达 | 说明 |
|---|---|---|
| 产品状态 | `已就绪` / `需要检查` / `同传中` | 放在顶栏右侧，避免大块状态卡。 |
| 音频源 | `Windows 系统声音`、`麦克风` | 使用下拉或两段式控件，不用大卡片。 |
| 语言方向 | `English → 中文` | 后续支持更多语言时改为下拉。 |
| 质量模式 | `实时优先`、`均衡`、`准确复盘` | 用户语言，不显示具体模型。 |
| 主操作 | `开始同传` | 页面唯一高权重按钮。 |
| 次操作 | `打开字幕窗预览` | 文字按钮或浅色按钮。 |
| 字幕预览 | 双语浮层预览 | 首页视觉中心。 |
| 状态摘要 | `已就绪 · 系统声音有输入 · 字幕窗可用` | 正常态一行展示。 |

### 首页禁止出现的信息

| 禁止项 | 原因 | 应放位置 |
|---|---|---|
| `Mock` | 开发者语义，会降低产品完成度 | 设置 / 开发者 |
| `ASR` | 模型层术语，普通用户不需要 | 高级 / 引擎详情 |
| `TTS` | 缩写不够产品化 | 普通设置用“语音播报”，模型细节进高级 |
| `模型路由` | 实现语言 | 高级 / 引擎详情 |
| `后端默认` | 工程配置语言 | 普通界面显示“自动” |
| `Voxtral + DeepSeek` | 可作为高级摘要，不应常驻首页 | 高级 / 引擎详情 |
| WebSocket / 事件调试 | 开发调试 | 高级 / 开发者 |

## 首页文案规范

| 工程文案 | 首页替代表达 |
|---|---|
| Agent 状态 | 同传服务 |
| Agent 已读取能力 | 同传服务已连接 |
| 模型路由 | 引擎 |
| 后端默认 | 自动 |
| ASR | 语音识别 |
| Translator | 翻译 |
| TTS | 语音播报 |
| Provider | 引擎 |
| Mock | 仅在开发者模式显示 |
| realtime error | 同传服务异常 |

首页正常态只保留用户状态摘要：

```text
已就绪 · 系统声音有输入 · 字幕窗可用
```

如需查看技术细节，用户主动进入高级 / 引擎详情后才显示：

```text
语音识别：Voxtral Realtime
翻译：DeepSeek
备用：FunASR
```

## 产品面分层

EchoSync 的三个主入口必须保持边界清楚：

```text
主控制面板
├─ 首页：开始同传
├─ 会议记录：历史记录、音频回放、双语复盘、导出
└─ 偏好设置：少量默认偏好

字幕窗口
├─ 实时双语字幕
├─ 轻工具栏
└─ 字幕外观设置

高级入口
├─ 引擎详情
├─ 故障处理
├─ 性能诊断
└─ 开发者
```

### 字幕窗口设置边界

字幕窗口设置只回答“这块字幕怎么显示、怎么放、是否可点”。它不能承载应用级配置。

允许出现：

```text
字幕窗口设置
├─ 字体大小
├─ 双语顺序
├─ 显示模式：双语 / 只看译文 / 只看原文
├─ 背景透明度
├─ 背景模糊
├─ 窗口阴影
├─ 锁定位置
├─ 鼠标穿透
└─ 重置位置
```

禁止出现：

```text
模型
引擎
Mock
WebSocket
日志
fallback
术语库管理
录制隐私
```

字幕窗口最多提供一个低权重入口：`更多偏好设置...`，点击后回到主控制面板的偏好设置，不在字幕窗口内展开复杂设置。

## 偏好设置信息架构

偏好设置应像工具偏好，而不是 AI pipeline 控制台。普通第一屏建议不超过 12 个可操作项。

```text
偏好设置
├─ 常规
│  ├─ 默认音频源
│  ├─ 默认语言方向
│  └─ 开始同传时打开字幕窗
├─ 字幕
│  ├─ 显示模式
│  ├─ 字号
│  ├─ 行数
│  └─ 位置
├─ 同传质量
│  ├─ 实时优先 / 均衡 / 准确优先
│  ├─ 语音播报：关闭 / 开启
│  └─ 引擎：自动
├─ 记录与隐私
│  ├─ 是否保存原始音频
│  ├─ 是否保存双语记录
│  └─ 自动清理周期
└─ 高级...
```

### 高级设置边界

高级设置不是常驻一级面板，而是低权重入口。只有用户主动进入时才显示模型、fallback、日志和 WebSocket。

```text
高级
├─ 引擎详情
│  ├─ 语音识别：自动
│  ├─ 翻译：自动
│  ├─ 语音播报：关闭
│  └─ 修正：自动
├─ 故障处理
│  ├─ 服务断开时自动重连
│  ├─ 翻译失败时先显示原文
│  └─ 识别失败时尝试备用引擎
├─ 性能诊断
│  ├─ 查看延迟统计
│  ├─ 导出诊断报告
│  └─ 开启详细日志
└─ 开发者
   ├─ Mock 引擎
   ├─ WebSocket 地址
   ├─ 原始事件流
   └─ 调试 payload
```

fallback 在普通设置里不能显示成链路编辑器。普通表达应是：

```text
故障时尽量继续生成字幕
```

展开后才显示：

```text
识别失败：尝试备用识别
翻译失败：先显示原文，恢复后补译
连接中断：自动重连
```

## 会议记录窗口设计

会议记录是 EchoSync 的长期价值面。实时字幕解决“当下听懂”，会议记录解决“事后复盘、校对、回放和导出”。它不属于设置，也不承担模型选择。

### 记录列表

记录列表采用轻量表格，而不是卡片瀑布流。截图中的方向是合理的：浅背景、白色行、充足行高、搜索入口、查看和删除操作。

```text
会议记录                                      搜索会议名称

序号    会议名称                  会议结束时间        会议时长      操作
1       2026年06月06日_记录_2      2026年06月06日 09:57  4分钟      查看 删除
2       2026年06月06日_记录_1      2026年06月06日 09:26  15分钟     查看 删除
3       2026年06月06日_记录        2026年06月06日 09:10  1分钟      查看 删除
```

列表第一阶段只保留必要信息：

| 信息 | 说明 |
|---|---|
| 会议名称 | 默认按日期生成，允许后续重命名。 |
| 会议结束时间 | 比开始时间更贴合“最近记录”排序。 |
| 会议时长 | 快速判断内容规模。 |
| 查看 | 进入双语复盘详情。 |
| 删除 | 删除本地记录，必须二次确认。 |

列表不展示模型、引擎、provider、WebSocket、详细延迟。性能诊断可以在详情页的高级信息中导出，但不能占据记录列表。

### 记录详情

详情页采用左右双栏：左侧原文，右侧译文。底部保留音频播放器，播放进度能定位到对应片段。

```text
返回    2026年06月06日_记录_2   内容自动保存 · 数据安全保护 · 译文由 AI 生成       字号  导出

┌ 原文 ───────────────────────┬ 译文 ───────────────────────┐
│ What are the constraints... │ 约束的集合是什么...          │
│                             │                              │
│ You someone is picking...   │ 拿起这个衣架...              │
│                             │                              │
│ 播放按钮  进度条  00:00/03:44│                              │
└─────────────────────────────┴──────────────────────────────┘
```

详情页必须保留这些能力：

| 能力 | 设计要求 |
|---|---|
| 原始音频回放 | 保留完整音频，不因实时 ASR 门控丢失静音和停顿。 |
| 双语对照 | 原文和译文并排，适合课程、会议和技术分享复盘。 |
| 时间对齐 | 片段携带起止时间；点击片段可以 seek 音频。 |
| 播放高亮 | 播放时高亮当前片段，但不让文字大幅跳动。 |
| 字号调整 | 只影响复盘阅读，不影响实时字幕窗口。 |
| 自动保存 | 文本修改后自动保存，避免导出前丢失。 |
| 导出 | 支持导出双语记录；后续再扩展 Markdown、SRT、DOCX。 |

### 记录详情禁止项

记录详情不是调试面板。默认不能出现：

```text
ASR provider
翻译 provider
Mock
WebSocket
原始事件流
模型路由
fallback chain
```

如果需要排查某条记录的延迟或质量，入口应是低权重的 `诊断信息`，并默认折叠：

```text
诊断信息
├─ 首字幕延迟
├─ 平均识别延迟
├─ 平均翻译延迟
├─ 字幕显示延迟
└─ 导出诊断报告
```

### 会议记录数据边界

会议记录的数据模型应服务复盘，而不是复刻实时事件流。

```text
SessionRecord
├─ id
├─ title
├─ startedAt
├─ endedAt
├─ durationMs
├─ audioUrl / audioPath
├─ segments[]
└─ exportState

Segment
├─ id
├─ startMs
├─ endMs
├─ sourceText
├─ translatedText
├─ revisionState
└─ isEdited
```

原始事件、provider payload、WebSocket frame 只进入开发者诊断，不进入记录详情主数据模型。

## 视觉方向

### 首页

- 背景：简洁纯白或极浅灰为主；如果采用深色半透明面板，只用于字幕预览和悬浮字幕，不让背景抢内容。
- 主面板：一个，不要多个大卡片。
- 主按钮：唯一高权重，建议蓝色或青蓝色。
- 字幕预览：尺寸要大，双语层级清楚；英文较小，中文较大。
- 状态摘要：细字、低对比、单行。
- 模型与状态诊断：默认收起，异常才展开。

### 偏好设置

- 使用简单列表和少量分组，不做复杂控制台。
- 第一屏只放用户能立即理解的默认偏好：音频源、语言、字幕、同传质量、记录与隐私。
- `引擎：自动` 是普通设置中唯一允许出现的引擎表达。
- 高级入口低权重显示，不默认展开。
- 不使用卡片网格式模型选择。

### 悬浮字幕预览

- 半透明深色背景，轻微毛玻璃。
- 8-12px 圆角，弱边框。
- 中文译文为主字号，原文为次字号。
- 鼠标聚焦时出现轻工具栏；默认不显示复杂控制。

### 会议记录

- 列表页采用浅背景和白色表格行，强调扫描和检索，不做重卡片。
- 详情页采用左右双栏，原文在左、译文在右，底部音频播放器固定可见。
- 顶部只保留返回、标题、自动保存/安全提示、字号、导出。
- 文本阅读区域要安静，不能用复杂渐变或装饰抢内容。
- 删除记录必须二次确认，导出按钮为主操作。

### 控件风格

- 音频源和目标语言用下拉行，不用模型卡片。
- 质量模式用三段式 segmented control。
- 开始同传用单一主按钮。
- 设置入口用轻量图标或文字按钮。

## 状态展示策略

### 正常态

```text
已就绪 · 系统声音有输入 · 字幕窗可用
```

### 异常态

异常时显示一条明确提示和修复入口：

```text
同传服务未连接
请先启动本地同传服务，或检查 8766 端口。
```

```text
系统声音暂无输入
请播放视频，或切换到麦克风。
```

```text
同传质量方案不可用
请在“偏好设置 / 高级”中检查服务或切换自动模式。
```

## Active 与 Finished 状态

本规范主要约束 Idle 首页，但 Active 和 Finished 也要延续同样语言：

- Active 不叫“会话驾驶舱”，可以叫“正在同传”。
- Active 的右栏不常驻展示模型名，只显示“质量模式：均衡”“延迟：xxx ms”“字幕窗：已打开”。
- Finished 应进入记录详情或生成记录草稿，重点是原始音频回放、双语记录、导出，不展示模型配置。
- 会议记录列表是独立入口，不嵌在偏好设置里。

## 非目标

- 本轮不重做悬浮字幕窗口交互细节。
- 本轮不新增真实模型供应商适配。
- 本轮不改变 Agent capabilities 协议。
- 本轮不移除开发者能力，只是从普通首页、普通偏好设置、字幕窗口设置和会议记录中移走。
- 本轮不做完整记录持久化实现；设计先明确会议记录列表和详情的产品边界。

## 验收标准

1. 首页不出现 `Mock`、`ASR`、`TTS`、`模型路由`、`provider`、`后端默认` 等开发术语。
2. 首页不出现模型选择卡片网格。
3. 首页只有一个视觉主按钮：`开始同传`。
4. 首页保留音频源、目标语言、质量模式、字幕窗预览、状态摘要。
5. 字幕预览是首页最大的信息视觉区域之一。
6. 普通偏好设置第一屏不出现 `ASR`、`TTS`、`provider`、`Mock`、`WebSocket`、`模型路由`。
7. 字幕窗口设置只出现字幕外观、位置、锁定和鼠标交互相关设置。
8. 正常状态只显示一行摘要；异常状态才展开诊断。
9. 视觉上避免后台管理系统风格：减少卡片数量、弱边框、充足留白、克制强调色。
10. 高级设置中才出现引擎详情、故障处理、性能诊断和开发者。
11. 会议记录列表只展示名称、结束时间、时长、查看、删除。
12. 会议记录详情默认只展示原文、译文、音频播放器、字号、导出和保存状态。
13. 会议记录详情不默认展示模型、WebSocket、原始事件流或 fallback chain。

## 实施建议

第一步只改 Idle 首页，把模型卡片迁走，保留现有能力预检逻辑但改成产品化摘要。

第二步把当前设置抽屉改成简单偏好设置，不在普通第一屏暴露模型名。现有 `ASR_PROVIDER_OPTIONS`、`TRANSLATION_PROVIDER_OPTIONS` 只进入高级设置。
