# EchoSync 主页与字幕弹窗需求设计

## 目标

将 EchoSync 首版 UI 从静态功能页升级为双入口产品体验：

- 主页是会话控制中心，负责事前启动、事中监控干预、事后复盘沉淀。
- 字幕弹窗是第二屏字幕舞台，负责低干扰阅读、轻量样式控制和可信状态表达。
- 首发默认体验偏向视频和网课，以系统音频加悬浮字幕压缩用户的 Aha Moment；远程会议和技术分享作为专业场景预设保留。

## 已对齐资料

- `doc/deep-research-report.md`：字幕优先、事件流、小窗口修正、术语表、延迟指标和可选 TTS。
- `doc/UI设计调研.md`：三栏工作台、剧场、阅读、紧凑悬浮、字幕状态机和动效克制原则。
- 当前 `apps/web`：已有工作台、剧场、阅读、紧凑悬浮模式和模拟事件流。
- 当前 `apps/desktop`：已有 Electron 主控窗口、透明置顶悬浮字幕窗、音频源目录和字幕状态机地基。

## 产品策略

采用“视频开箱 + 专业驾驶舱骨架”路线。

首次打开时，产品默认把用户引向低风险、容易验证效果的视频或网课场景。默认路径应优先让用户选择或确认系统音频，打开字幕弹窗，并在视频播放时立即看到可读译文。会议、技术分享和文件回放不隐藏，但作为场景预设存在。

这个策略的目标是“第一次用很顺，第二次用发现它不是玩具”。首版必须同时保留专业驾驶舱骨架：健康度、转写回溯、术语快加和导出前清理。

## 主页定位

主页是 Stateful Hybrid 会话控制中心。它不是营销首页，也不是静态功能宫格。主页应保持同一套空间骨架，但视觉重心随会话生命周期变化。

生命周期：

```text
Idle -> Active -> Finished
```

稳定骨架：

- 顶部状态栏：品牌、会话状态、紧凑控制、字幕窗状态。
- 中心主任务区：随生命周期切换主要任务。
- 右侧驾驶舱栏：准备检查、健康度、术语和质量摘要。
- 底部区域：最近记录、事件提示或本次摘要入口。

## Idle 态

Idle 态的目标是快速起飞和找回最近记录。

中心主任务区：

- 默认场景为视频/网课。
- 平台感知默认输入源：Windows 桌面端优先系统音频；Web 或无系统音频能力的平台优先标签页音频或麦克风。
- 主 CTA 是“开始同传”，旁边提供“打开字幕弹窗预览”。
- 语言方向放在主区上方，默认 `English -> 中文`，支持快速互换。
- 场景预设包含视频/网课、远程会议、技术分享、文件回放。视频/网课默认高亮。

起飞前校验：

- 主 CTA 附近必须有 Pre-flight Audio Visualizer，表现为实时电平条或微型波形。
- 切换输入源后，用户应能通过说话或播放视频看到电平变化。
- 准备检查不应只是静态图标，至少要真实反映音频活动、权限、provider 可用性和字幕窗可打开状态。
- 若没有检测到音频活动，CTA 仍可点击，但要给出非阻断提醒。

右侧驾驶舱栏：

- Idle 态显示准备就绪检查：音频源、ASR provider、翻译 provider、字幕弹窗状态、权限状态。
- 提供轻量设置入口：字幕样式、术语库、模型/延迟策略。

底部区域：

- 展示最近会话列表。
- 每条记录展示时间、场景、时长、语言方向、是否已导出。
- 最近会话可一键进入复盘。

## Active 态

Active 态的目标是监控和干预，不重复字幕弹窗的阅读职责。

顶部状态栏：

- 启动区折叠为紧凑控制条。
- 保留暂停、停止、输入源、电平、字幕窗显示状态、鼠标穿透状态。
- 会话状态至少包含 Listening、Recovering、Provider Error、No Audio。

中心主任务区：

- 主体是最近 3-5 分钟转写历史。
- 每个片段展示原文、译文、状态、时间戳和修订标记。
- 支持复制单句、锁定片段、跳到当前、查看最近修订。
- 当前片段可弱高亮，但不得大面积闪动。

Auto-scroll Lock：

- 用户向上滚动超过阈值、悬停历史片段、选择文本、打开复制/锁定操作时，自动滚动必须立即暂停。
- 暂停期间新片段和 patch 继续进入数据流，但不得强行把视图拉回底部。
- 底部显示“有新内容，回到当前”提示。
- 用户点击提示后恢复自动滚动并跳到当前片段。

右侧驾驶舱栏：

- 健康度指标包括首字幕延迟、稳定提交延迟、ASR RTF、空片段率、patch 次数、音频电平。
- 术语快加支持当前会话临时术语，格式为 `source -> target`。
- 模式策略支持低延迟、平衡、准确优先，只改变 pipeline 参数，不改变页面结构。

Term Syncing：

- 新增术语后必须显示状态：注入中、已生效、失败/重试。
- UI 必须管理用户预期：新术语通常从后续片段开始影响翻译，不承诺立即改写当前正在生成的句子。
- 若术语触发 patch，可在对应片段显示修订原因。

Uncertainty / Calibration：

- 若底层模型提供 token 或片段置信度，转写历史应能弱标记低置信专名或片段。
- 若底层不提供原生置信度，只能基于稳定度、patch 频率、空片段率推断置信水平，并在 UI 文案上避免伪装成模型原生置信度。
- 健康度面板可显示当前置信水平、稳定度或不确定性提示。

底部区域：

- 展示轻量事件日志：provider 重连、术语生效、音频源中断、片段锁定。
- 不做开发者控制台，不展示过多原始调试事件。

## Finished 态

Finished 态的目标是趁热沉淀，不生硬回到 Idle。

中心主任务区：

- 展示本次会话快速复盘卡：时长、总字数、片段数、修订次数、平均延迟。
- 提供导出入口：Markdown 笔记、SRT 字幕、TXT 双语转写。
- 展示本次高频术语建议，允许加入永久术语库。

Pre-export Edit：

- 导出前提供“快速清理”选项。
- 用户可在最终 transcript 中双击修正明显错词、人名、术语。
- 首版不做完整富文本编辑器，只做导出前轻量校对。

右侧驾驶舱栏：

- 展示本次质量摘要：延迟是否稳定、是否长时间无音频、哪些术语触发过修订。
- 提供“继续新会话”和“回到最近记录”。

底部区域：

- 本次完整转写入口。
- 最近会话列表更新，并将刚结束的会话置顶。

## 字幕弹窗定位

字幕弹窗是第二屏字幕舞台，不是迷你控制台。它长期覆盖在视频、会议、IDE 或浏览器上方，负责伴随式阅读和轻量样式控制。

弹窗不承担：

- 术语库编辑。
- 完整转写历史。
- Provider 健康度。
- 导出。
- 复杂模型参数。

## Layer A：默认极简态

默认态只显示当前译文，必要时显示一行弱化原文。

核心规则：

- 默认启用鼠标穿透，点击事件落到下层应用。
- 不显示按钮、菜单、指标和术语表。
- 显示译文、可选原文和轻量状态点。
- `interim` 文本用低透明度表达未稳定。
- `stable` 文本用满对比度显示。
- 文本更新不能造成字幕条大幅跳动。
- 无音频时显示极轻提示“等待音频”，几秒后弱化。

## Layer B：Hover 轻控制态

Hover 态只解决字幕本身的快速调整。

Hover Intent：

- 鼠标停留 150-250ms 后才唤醒 Layer B。
- 快速划过字幕区域时，保持 Layer A 穿透态，不弹出控制栏。
- 唤醒后才拦截鼠标事件。
- 用户拖拽、调整大小或打开轻菜单时，不自动收起。

可见控制：

- 字号增减。
- 单语/双语切换。
- 透明度/背景强度。
- Pin 驻留。
- 暂停字幕显示或隐藏字幕窗。
- 鼠标穿透状态提示。
- 拖拽手柄和尺寸调节边界。

收起规则：

- 鼠标离开一小段时间后自动回到默认态。
- 控件不得压住字幕主文本。

## Layer C：Pin 小型双语舞台

Pin 态用于认真跟读或第二屏观看。它显示最近 2-3 条字幕片段，但仍比主页轻。

内容结构：

- 最近 2-3 行译文。
- 可选每行原文。
- 当前行更大，历史行更弱。
- 每行状态：`Interim / Stable / Revised / Locked`。
- 低置信或不确定专名可用轻微下划线提示。

可信状态表达：

- `Interim`：正在确认中，透明度低，不进入稳定历史。
- `Stable`：可读主文本，但仍可能接受小窗口修订。
- `Revised`：只高亮变化片段，并可短暂显示术语或上下文修正原因。
- `Locked`：不再自动变化，视觉安静。

Visual Decay：

- `Revised` 高亮和修正原因必须有时间衰减。
- 建议 1.5-2 秒后平滑过渡回 Stable 标准样式。
- 频繁 patch 不得形成长期彩色堆叠。

轻动作：

- 复制当前译文。
- 锁定当前片段。
- 打开主页并定位到该片段。
- 取消 Pin 回到默认态。

## 弹窗尺寸与边缘策略

- 从 A 切到 C 时，如果空间不足，优先向屏幕内侧展开。
- 靠近底边时向上展开，靠近右边时向左扩展。
- 最小尺寸保证一行字幕可读。
- 最大尺寸不能超过屏幕安全区域。
- 记住用户上次位置、宽度、Pin 状态和透明度。
- 默认态不拦截点击；Hover 后显示拖拽手柄。
- 拖拽时暂停穿透。
- 拖拽结束后按用户设置恢复穿透或交互状态。

## 弹窗唤醒后备方案

桌面端透明穿透窗口可能无法可靠捕获 hover。必须设计系统级 fallback wake-up：

- 提供全局快捷键作为强制唤醒或锁定 Layer B 的后备方案。
- 推荐默认候选：`Ctrl+Shift+S` 或用户可配置快捷键。
- 若 OS 窗口穿透导致 hover 不可用，用户仍能通过快捷键唤醒轻控制。
- 主控主页必须有“一键召回字幕窗”能力，处理窗口移出屏幕或失焦。

## 异常与恢复

- 无音频：弹窗显示轻提示，主页负责详细诊断。
- 连接恢复中：弹窗显示状态点，不展开大错误。
- Provider 错误：弹窗显示短提示，并引导回主页查看。
- 字幕窗移出屏幕：主页可召回居中。
- 音频设备断开：由 `AudioContextManager` 派发事件，主页提示并尝试恢复。

## 组件边界

显式组件：

- `HomeShell`：管理 `idle | active | finished` 生命周期。
- `SessionStartPanel`：Idle 快速开始。
- `PreflightAudioVisualizer`：起飞前音频电平校验。
- `ActiveControlBar`：Active 顶部紧凑控制。
- `TranscriptMonitor`：Active 转写回溯、滚动锁和片段操作。
- `HealthPanel`：延迟、RTF、音频、电平、patch、置信水平。
- `TermQuickAdd`：当前会话术语快加和同步状态。
- `SessionSummary`：Finished 快速复盘。
- `PreExportEditor`：导出前轻量清理。
- `RecentSessions`：Idle/Finished 共用。
- `OverlayLauncher`：控制字幕弹窗显示、隐藏、穿透、Pin、召回。
- `SubtitleOverlay`：弹窗分层容器。
- `OverlayCaptionLine`：弹窗字幕行和状态表达。

隐形逻辑组件：

- `AudioContextManager`：管理音频权限、设备切换、系统音频失败、耳机拔出、静音输入等底层状态，并向主页派发事件。
- `GlobalToaster`：处理 provider 瞬时超时、自动重连、术语同步成功、音频源中断等非阻断提示。
- `OverlayInteractionManager`：管理穿透、hover intent、fallback wake-up、拖拽和 Pin 状态。
- `SessionLifecycleController`：把音频、provider、字幕事件和 UI 生命周期收束为 `Idle -> Active -> Finished`。

## 事件与数据要求

现有字幕事件可以继续使用：

- `translation.partial`
- `translation.patch`
- `segment.commit`

前端 UI 还需要派生或补充这些 UI 状态：

- 音频活动状态：silent、active、clipping、device_missing、permission_denied。
- 术语同步状态：draft、syncing、active、failed。
- 自动滚动状态：following、locked、new_content_available。
- 弹窗交互状态：pass_through、hover_pending、interactive、pinned、dragging。
- 会话生命周期：idle、active、finished。
- 置信显示来源：native_confidence、derived_stability、unavailable。

## 非目标

首版不做：

- 完整富文本编辑器。
- 完整会议纪要自动写作系统。
- 多窗口复杂布局管理。
- 术语库批量管理和企业权限。
- Provider 深度调参 UI。
- 在弹窗内展示完整健康度或完整历史。

- 首发默认视频/网课，但没有封死会议和技术分享扩展。
- 明确处理了音频校验、滚动锁、术语同步、置信度来源、导出前清理、hover intent、视觉衰减和穿透唤醒后备方案。
