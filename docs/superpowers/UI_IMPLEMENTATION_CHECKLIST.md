# EchoSync 前端重构实施清单

基于 [首页同传启动器、偏好设置与会议记录设计规范](specs/2026-06-06-home-launcher-engine-settings-design.md)

## 重构目标

将 EchoSync Desktop 从"模型控制台"收敛为三个清晰产品面：
- **主控制面板**：负责开始同传
- **字幕窗口**：负责覆盖式实时字幕
- **会议记录**：负责回看、校对、播放和导出

## 实施阶段

### 阶段 1：首页重构（Idle 状态）

#### 1.1 移除开发术语
- [ ] 移除首页的 `Mock` 相关显示
- [ ] 移除 `ASR`、`TTS`、`Agent`、`provider` 等术语
- [ ] 将 `模型路由` 改为 `引擎`
- [ ] 将 `后端默认` 改为 `自动`
- [ ] 将工程术语替换为用户友好表达（参考规范中的文案规范表）

#### 1.2 简化首页布局
- [ ] 移除模型选择卡片网格
- [ ] 创建单一主面板布局
- [ ] 保留核心控件：
  - [ ] 音频源选择（下拉或两段式控件）
  - [ ] 语言方向选择（`English → 中文`）
  - [ ] 质量模式选择（三段式 segmented control：`实时优先 | 均衡 | 准确复盘`）
- [ ] 设计并实现双语字幕预览区域（首页视觉中心）
- [ ] 创建单一高权重主按钮：`开始同传`
- [ ] 添加次操作按钮：`打开字幕窗预览`（文字或浅色按钮）

#### 1.3 状态展示优化
- [ ] 实现正常态状态摘要：`已就绪 · 系统声音有输入 · 字幕窗可用`
- [ ] 实现异常态状态展开逻辑
- [ ] 设计异常提示卡片样式（包含问题描述和修复入口）
- [ ] 移除能力预检的详细技术信息展示
- [ ] 将状态显示改为顶栏右侧轻量标识：`已就绪` / `需要检查` / `同传中`

#### 1.4 视觉调整
- [ ] 应用简洁背景（纯白或极浅灰）
- [ ] 优化字幕预览样式：
  - [ ] 半透明深色背景
  - [ ] 轻微毛玻璃效果
  - [ ] 8-12px 圆角
  - [ ] 双语层级清晰（英文较小，中文较大）
- [ ] 确保首页只有一个视觉主按钮
- [ ] 减少边框强度，增加留白

## 实施策略

### 渐进式重构
1. 先在新分支实现新的组件结构
2. 保持现有功能可用的同时逐步替换
3. 使用特性开关控制新旧 UI 切换（可选）

### 数据迁移
1. 确保旧设置数据能迁移到新结构
2. 提供默认值处理逻辑
3. 向后兼容 Agent capabilities 协议

### 文档同步
1. 更新用户文档
2. 更新开发者文档
3. 记录重大变更

## 验收标准（来自设计规范）

1. ✅ 首页不出现 `Mock`、`ASR`、`TTS`、`模型路由`、`provider`、`后端默认` 等开发术语
2. ✅ 首页不出现模型选择卡片网格
3. ✅ 首页只有一个视觉主按钮：`开始同传`
4. ✅ 首页保留音频源、目标语言、质量模式、字幕窗预览、状态摘要
5. ✅ 字幕预览是首页最大的信息视觉区域之一
6. ✅ 普通偏好设置第一屏不出现 `ASR`、`TTS`、`provider`、`Mock`、`WebSocket`、`模型路由`
7. ✅ 字幕窗口设置只出现字幕外观、位置、锁定和鼠标交互相关设置
8. ✅ 正常状态只显示一行摘要；异常状态才展开诊断
9. ✅ 视觉上避免后台管理系统风格：减少卡片数量、弱边框、充足留白、克制强调色
10. ✅ 高级设置中才出现引擎详情、故障处理、性能诊断和开发者
11. ✅ 会议记录列表只展示名称、结束时间、时长、查看、删除
12. ✅ 会议记录详情默认只展示原文、译文、音频播放器、字号、导出和保存状态
13. ✅ 会议记录详情不默认展示模型、WebSocket、原始事件流或 fallback chain

## 注意事项

### 保持兼容
- Agent capabilities 协议不变
- 后端 API 接口不变
- 本地数据存储格式向后兼容

### 不在本轮实施
- 悬浮字幕窗口交互细节重做
- 新增真实模型供应商适配
- 移除开发者能力（只是隐藏）
- 完整记录持久化实现（先明确产品边界）
