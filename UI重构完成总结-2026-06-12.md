# UI 重构审计与提交说明 - 2026-06-13

## 范围

本次提交收敛 `apps/desktop` 桌面端前端结构，目标是降低 `main.tsx` 的耦合度，稳定字幕窗口、控制台、会话复盘和偏好设置的边界。

本次不纳入 Agent 翻译服务改动；当前工作区里 `apps/agent/src/echosync_agent/services/translation/deepseek_translator.py` 仍是独立脏文件，提交时应继续避开。

## 已完成

### 组件边界

* 从 `main.tsx` 抽出字幕窗口、控制台首页、会话状态、复盘记录、偏好设置、字幕样式面板等组件。
* 复盘详情页拆成 `RecordDetailHeader`、`RecordPlayer`、`TranscriptToolbar`、`TranscriptSegment`、`SummaryPanel` 等专用组件。
* 新增通用组件 `TabBar`、`SearchField`、`WaveformProgress`、`ToolbarIcon`、`GradientButton`、`GlassCard`，避免业务组件重复实现基础 UI。

### Hook 与工具

* 新增 `useSessionPreferencesState`，把会话偏好加载、同步和更新逻辑从主渲染入口移出。
* 新增 `useSharedSubtitleStyle`、`useCompleteCaptionItemVisibility`，收敛字幕样式同步和完整字幕可见性逻辑。
* 新增复盘播放、草稿生成、页面标题、时间线布局等纯工具，并补充对应单元测试。

### IPC 边界

* 渲染层对 `window.echosyncDesktop` 的访问集中在 `apps/desktop/src/renderer/services/ipc/*`。
* 业务组件通过 `services/ipc` 调用采集、实时字幕、会议记录、剪贴板、窗口控制和字幕样式能力。
* 审计未发现前端新增 provider secret、API key 或 SDK 直连。

### 字幕与复盘确定性

* `caption-store` 与 `caption-text-view` 增加回归测试，覆盖最终字幕提交文本、稳定文本保护、修订文本和锁定行为。
* `segment.commit` 使用最终 target 文本，同时保留更长的 source 文本保护，避免双语复盘记录丢失最终译文。
* 文本收缩阈值收敛到 `text-protection-constants.ts`，减少隐式魔法值。

### 复盘详情交互审计修复

* 复盘详情页的标签页、搜索、上一处/下一处、音量、倍速已接入真实 React 状态。
* 移除详情页占位 `console.log` 和未实现阅读设置入口。
* 搜索高亮增加正则转义，搜索 `[`、`?`、`.` 等字符不会破坏渲染。
* 音频元素会同步应用当前音量和倍速，元数据就绪、跳转补偿和播放恢复路径均保持一致。

### 样式组织

* `styles.css` 改为唯一入口文件，按责任引入 `tokens.css`、`base.css`、`shell.css`、`home.css`、`session-records.css`、`glass-design.css`、`session.css`、`caption.css`、`style-panel.css` 等文件。
* 字幕窗口与会话页面样式继续下钻到 `styles/caption/*`、`styles/session/*`，避免继续堆大单文件。
* 复盘详情样式改为 `record*` 域名前缀，避免 `.summaryPanel`、`.panel`、`.toolbar`、`.segment` 等泛类与会话页或字幕页互相覆盖。
* 视觉契约测试通过 `styles.css` 入口递归读取 import，覆盖真实运行样式路径。
* 保持桌面工具产品的安静、密集、可扫读方向，没有引入营销页式布局。
